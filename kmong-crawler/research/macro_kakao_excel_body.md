### [H2] 자주 묻는 질문 (FAQ)

**Q. 카카오톡 API 사용료는 별도인가요**
알림톡 1건 발송에 9~13원이 카카오에 별도로 청구됩니다 (사장님 카카오 비즈머니에서 차감).
저희가 받는 금액은 시스템 구축비이며, 운영 중 발생하는 발송비용은 사장님이 직접 정산하시는 구조입니다.
월 1,000건 발송 기준 약 12,000원 수준입니다.

**Q. 카카오 정책 변경되면 시스템이 죽지 않나요**
유지보수 기간 동안은 저희가 무상으로 대응합니다.
유지보수 종료 후에도 정책 변경 알림 수신 시 사장님께 우선 안내드리고, 추가 작업이 필요하면 별도 견적으로 진행합니다.

**Q. 친구톡, 알림톡, 오픈채팅 무엇이 다른가요**
알림톡은 사장님 사업자명으로 발송하는 공식 메시지 (예: 결제완료 안내).
친구톡은 카카오 채널 친구로 추가된 분께 보내는 마케팅 메시지.
오픈채팅은 익명 채팅방 자동발송.
DELUXE 이상에서 셋 다 함께 운영 가능합니다.

**Q. 사장님 PC가 꺼지면 발송이 멈추나요**
저희 권장 구성은 클라우드 서버 (월 5,000원 수준) 또는 사장님 PC 24시간 켜두기 중 선택입니다.
PREMIUM은 클라우드 서버 1년치 (60,000원) 포함입니다.

**Q. 다른 업체 봇이랑 뭐가 다른가요**
대부분의 경쟁 서비스가 알림톡만, 또는 친구톡만 단일 채널이거나, 트리거가 1~2개로 제한됩니다.
저희는 **카카오 3채널 + 트리거 무제한 + 양방향 응답수신 + 노코드 운영 GUI**까지 한 번에 갖춥니다.

## 📌 작업 절차 (#DESCRIPTION_PROGRESS)

### [H2] 작업은 이렇게 진행됩니다

1. **인콰이어리 / 견적** — 24시간 내 사장님 업종에 맞춘 견적서 회신
2. **시나리오 정의 미팅** — 30분 카톡·화상으로 발송 시점/대상/문구 확정 (무료)
3. **알림톡 템플릿 등록 대행** — 카카오 사업자 인증 후 템플릿 1~3건 등록 (DELUXE 이상은 1건 무료)
4. **트리거 소스 연결** — 구글시트, 노션, 자체 DB, 웹훅 등 사장님이 쓰시는 도구 연결
5. **테스트 발송 검증** — 사장님 본인 카톡으로 5건 테스트 후 발송 안정성 확인
6. **인계 / 매뉴얼 / 무상 유지보수 시작** — PDF 사용설명서 + 30~180일 유지보수 (패키지별)

각 단계 완료마다 카톡으로 진행 보고드립니다.
사장님은 시나리오 정의 미팅 30분 외에는 따로 시간 내실 일이 없습니다.

## 📌 의뢰 전 준비사항 (#DESCRIPTION_PREPARATION)

### [H2] 미팅 전에 이것만 정리해두시면 빠릅니다

- **카카오톡 비즈니스 채널** 보유 여부 (없으면 등록 대행 가능)
- **사업자등록증 사본** (알림톡 템플릿 등록에 필요)
- **현재 보내고 계신 메시지 예시** (3~5개 정도 카톡 또는 메모로 정리)
- **발송 트리거가 무엇인지** — 예: 결제 완료 시 / 매일 오전 9시 / 시트에 행 추가 시
- **수신자 정보를 어디에 보관하시는지** — 구글시트, 노션, 엑셀 파일, 자체 시스템 등

위 5가지가 정리돼 있으면 시나리오 정의 미팅 30분 안에 모든 결정이 끝납니다.
정리가 어려우시면 미팅에서 함께 정리해드리니 그냥 오셔도 됩니다.

## 📌 크몽 FAQ 탭 (#FAQ — 행정/환불/계산서, 본문 FAQ와 중복 금지)

**Q. 작업 중간 취소가 가능한가요**
크몽 정책에 따라 진행률에 비례한 환불이 가능합니다.
시나리오 정의 미팅 전이면 100% 환불, 알림톡 등록 단계 후에는 50% 환불, 테스트 단계 후에는 20% 환불입니다.

**Q. 세금계산서 발행되나요**
사업자 회원 결제 시 자동 발행됩니다.
개인 회원 결제 후 사업자 전환 처리도 가능합니다 (결제 후 30일 이내).

**Q. 작업 완료 후 추가 수정 비용은**
패키지별 무상 유지보수 기간 동안은 시스템 오류 / 카카오 정책 대응 / 트리거 미세조정은 무료입니다.
신규 채널 추가, 트리거 신규 추가, 메시지 디자인 전면 개편은 별도 견적입니다.

**Q. 결과물 소스코드 공개되나요**
사장님 PC 또는 사장님 클라우드 계정에 직접 설치되며 소스 그대로 인계됩니다.
저희 서버에 의존하지 않으므로 폐업·종속 리스크가 없습니다.
