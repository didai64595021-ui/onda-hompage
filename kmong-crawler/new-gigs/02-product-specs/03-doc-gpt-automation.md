### 3. 자동화 가능 업무 예시 (Before → After)
| 업무 | Before (수작업) | After (자동화) |
|---|---|---|
| 송장 처리 | PDF 열기 → 항목 확인 → 엑셀 입력 (건당 5분) | PDF 업로드 → GPT 자동 추출 → 엑셀 자동 입력 (건당 3초) |
| 주간 보고서 | 데이터 수집 → 분석 → 문서 작성 (3시간) | 자동 수집 → GPT 분석·요약 → 보고서 생성 (0분) |
| 이메일 분류 | 메일 확인 → 내용 파악 → 담당자 전달 (건당 2분) | 수신 즉시 → GPT 분류 → 담당자 자동 전달 (즉시) |
| 고객 문의 요약 | 내용 읽기 → 요약 → 공유 (건당 10분) | GPT 자동 요약 → 슬랙/시트 자동 기록 (즉시) |
| 견적서 생성 | 단가표 조회 → 수량 계산 → 포맷 맞추기 (30분) | 견적 요청 이메일 → GPT 파싱 → 자동 생성 (30초) |

### 4. 왜 AI 자동화인가? (RPA vs AI 비교)
- 기존 RPA: 정형 데이터만 처리 (엑셀 셀 위치 고정, HTML 태그 고정)
- AI 자동화: **비정형 문서(PDF 이미지, 손글씨, 자유 양식)도 GPT가 이해하고 처리**
- GPT 결합으로 "판단이 필요한 업무"까지 자동화
- Make/n8n = 드래그&드롭 → 코딩 없이 누구나 수정 가능

### 5. 차별화 포인트
- ✅ **노코드 + AI** — Make/n8n으로 구축하므로 비개발자도 직접 수정 가능
- ✅ **다년간 자동화 실전 경험** — 광고 자동화·데이터 파이프라인·AI 챗봇 5종 운영 중
- ✅ **GPT + Claude 이중 활용** — 한국어는 Claude, 범용은 GPT-4 — 최적 모델 배치
- ✅ **100% 소스 양도** — 모든 패키지 소스코드/워크플로우 완전 양도
- ✅ **n8n 자체호스팅** → 월 0원 운영 가능 (SaaS 구독료 없음)

## FAQ (10개)

**Q1. 코딩을 몰라도 나중에 수정할 수 있나요?**
A. **네. Make/n8n은 드래그&드롭 노코드 도구**입니다. 교육 영상 제공 후 90% 이상 직접 수정 가능합니다.

**Q2. 어떤 업무가 자동화 가능한가요?**
A. 반복적인 데이터 입력, 문서 변환, 분류, 요약, 알림 전송, 리포트 생성 등 **"반복"과 "판단"이 있는 모든 업무**가 대상입니다. 무료 상담에서 가능 여부를 판단해 드립니다.

**Q3. GPT API 비용은 얼마인가요?**
A. 일반 사무 자동화 기준 **월 1~10달러** 수준입니다 (GPT-4o 기준 1,000건/월). 사전에 예상 비용을 안내합니다.

**Q4. 기존 ERP/그룹웨어와 연동 가능한가요?**
A. API가 있는 시스템은 대부분 가능합니다 (SAP, 더존, 얼마에요 등). API 없는 경우 Python 크롤러로 대체 가능합니다.

**Q5. 보안이 걱정됩니다. 사내 문서가 외부로 나가나요?**
A. OpenAI/Anthropic 비즈니스 API는 **데이터를 모델 학습에 사용하지 않습니다**. 추가로 n8n 자체호스팅을 쓰면 **데이터가 외부 SaaS를 거치지 않습니다**. 온프레미스 Add-on도 가능합니다.

**Q6. Make 무료 플랜으로도 운영 가능한가요?**
A. Make 무료 플랜(월 1,000 오퍼레이션)으로 소규모 가능합니다. n8n 자체호스팅은 **완전 무료**(오퍼레이션 무제한)이므로 비용 부담이 0원입니다.

**Q7. 처리 속도는 어떤가요?**
A. 문서 1건당 3~10초 수준입니다. 하루 수천 건 처리도 자동 스케줄로 가능합니다.

**Q8. 여러 워크플로우를 한꺼번에 의뢰할 수 있나요?**
A. Standard 1개, Deluxe 3개, Premium 무제한입니다. 한 번에 여러 업무를 자동화하면 단가가 낮아지므로 **Deluxe 이상 권장**합니다.

**Q9. 기존 Zapier/Make를 이미 쓰고 있는데 확장 가능한가요?**
A. 기존 워크플로우에 AI 노드를 추가하거나, n8n으로 마이그레이션 가능합니다.

**Q10. 환불이 가능한가요?**
A. 작업 착수 전 100% 환불, 착수 후에는 크몽 환불 정책을 따릅니다.
