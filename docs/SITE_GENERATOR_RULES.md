# 사이트 생성 규칙 — Award 수상작 기반 디자인만

## 🚨 템플릿형 절대 금지 (2026-03-13, 영구, 최최우선)
**아래 패턴이 하나라도 해당되면 = 템플릿 = 즉시 폐기 + 재제작:**
1. **뻔한 섹션 순서**: 히어로→서비스카드→팀소개→카운터→후기→CTA 반복
2. **균일한 카드 그리드**: 3열 카드 반복, 동일 높이/동일 구조
3. **장식용 CSS 애니메이션만 있고 실제 인터랙션 없음**: hover시 살짝 올라가기, fadeIn만 반복
4. **한 파일에 모든 스타일 인라인**: 공통 CSS 없이 각 HTML에 <style> 복붙
5. **Unsplash 이미지만 깔고 내용 빈약**: 이미지로 때우고 텍스트/기능 부실
6. **모든 페이지 레이아웃 동일**: 헤더-본문-푸터 외 구조적 차별화 없음
7. **버튼/링크가 장식**: 클릭해도 아무 동작 없는 버튼

**Award급 = 다음 필수 충족:**
- 페이지마다 다른 레이아웃 구조 (스플릿, 매거진, 벤토 등 혼합)
- 실제 동작하는 인터랙션 최소 5개 (필터, 모달, 탭, 아코디언, 슬라이더, 드래그 등)
- 마이크로 인터랙션 (버튼 ripple, 커서 이펙트, 스크롤 트리거 등)
- 타이포그래피 위계 명확 (H1~H4 크기/무게 차별화, 본문과 구분)
- 여백 설계 (섹션 간 120px+, 요소 간 8px 그리드)
- 컬러 운용 (배경/텍스트/액센트/서브 4단계 이상)
- 각 페이지 고유 히어로/진입점 (전부 같은 히어로 금지)

## ⚠️ 최우선 원칙 (2026-03-12, 영구)
1. **모든 사이트는 실제 Awwwards/FWA/CSS Design Awards 수상작을 레퍼런스로 삼는다**
2. **수상작의 레이아웃·타이포·인터랙션·여백·컬러 운용법을 분석 → 업종에 맞게 적용**
3. **아래 디자인 풀은 참고용일 뿐, 풀에서 조합 돌리는 "템플릿 생성" 절대 금지**
4. **제작 전 반드시 해당 업종 + award winning site 웹 검색 → 3개 이상 레퍼런스 확보**
5. **레퍼런스 사이트의 "이 부분이 좋다"를 구체적으로 명시 후 적용**
6. 뻔한 카드 나열, 단조로운 그리드, 어설픈 그라데이션 = 저품질 = 즉시 폐기

## 필수 기본 요건 (모든 사이트, 영구)
1. **헤더/푸터 통일** — 전 페이지 동일 NAV + 동일 FOOTER (CSS/HTML 완전 일치)
2. **모바일 반응형** — 375px / 480px / 768px / 1024px / 1440px 전부 대응
3. **디바이스 크로스체킹** — 완료 전 각 브레이크포인트 검증 (레이아웃 깨짐, 오버플로우, 텍스트 잘림, 터치타겟 48px)
4. **페이지당 최소 1,500줄** — showcase-apartment 수준 밀도
5. **공통 styles.css 분리** — 헤더/푸터/변수/리셋은 공통 파일로
