■ 1. 시스템 개요
────────────────────────────────────────────────────────────────────────────────
- 네이버 플레이스 광고 로직 모니터링 대시보드
- 기술 스택: Next.js 14 + Supabase + Tailwind CSS + Vercel
- 데이터 소스: adlog.kr (애드로그 크롤링)
- AI 분석: Claude API (Anthropic)
- 대상: 온다마케팅 내부 직원용

■ 2. 크롤링 로직
────────────────────────────────────────────────────────────────────────────────
[데이터 수집 대상]
- adlog.kr > 플레이스 순위 체크 페이지
- 그룹이 "진행중" 또는 "노출중"인 업체만 관리 대상 (is_managed=true)
- 현재: 관리 업체 45개, 관리 키워드 26개

[크롤링 방식]
- crawler/adlog.ts — fetch API 기반 (Puppeteer 없음)
- 로그인: adlog.kr 세션쿠키(adsession) 기반 인증
- 일일 크롤링 (daily):
  · ajax_place_list.php API 호출 (키워드별)
  · HTML 응답 파싱 → 순위, N1, N2, N3, 블로그리뷰, 방문자리뷰, 저장수 추출
  · daily_snapshots 테이블에 UPSERT
  · 전일 N2와 비교하여 n2_change 자동 계산
- 히스토리 크롤링 (history):
  · ajax_place_keyword.php API 호출 (업체별 30일치)
  · 배치 단위 + 체크포인트 저장 (중단 후 재개 가능)

[N1 / N2 지표]
- N1 = SEO 점수 (저품질/약저품질 판별)
- N2 = 순위 결정 핵심 점수
- N2 변동이 -0.01 이하 → YELLOW 경고
- N2 변동이 -0.03 이하 → RED 경고
- 순위 5위 초과 → YELLOW, 7위 초과 → RED

[크롤링 스케줄]
- adlog 데이터 초기화: 매일 02:00~04:00 KST
- 크롤링 윈도우: 16:00 이후 실행 권장
- 재시도: 최대 3회, 1시간 간격
- 키워드 간 딜레이: 3초 / 업체 간: 2초 (차단 방지)

[크롤링 실패 대응]
- 실패건 → crawl_failed_items 테이블 기록
- 직원/관리자 화면에서 미해결 실패건 노출 → 수동 입력 유도
- evidence_type: 'auto'(크롤링) / 'manual'(수동입력) 구분

■ 3. 대시보드 (직원용)
────────────────────────────────────────────────────────────────────────────────
[메인 대시보드] /dashboard
- 요약 카드: 관리 키워드 수, 관리 업체 수, RED 경고, YELLOW 경고
- 업체 카드: 순위, N2 점수, N2 변동, N2 추이 스파크라인
- 블로그리뷰/방문자리뷰/저장수 표시
- 업체 클릭 → 상세 모달 (동일키워드 경쟁업체, 수익성, 고객 특이사항)

■ 5. 인증 시스템
────────────────────────────────────────────────────────────────────────────────
- Supabase Auth 기반
- 역할: admin (관리자) / staff (직원)
- 세션 체크: 5초 타임아웃 → 실패 시 자동 로그인 리다이렉트
- 로그인 페이지: 3초 후 재시도 버튼 표시
- 비밀번호 변경 기능
- 관리자만 /admin 접근 가능 (AuthGuard)

■ 6. DB 테이블 구조 (주요)
────────────────────────────────────────────────────────────────────────────────
- users: 사용자 (email, name, role, error_rate)
- keywords: 키워드 (keyword, difficulty_grade, is_managed)
- businesses: 업체 (name, keyword_id, place_id, is_managed, contract_fee)
- daily_snapshots: 일일 스냅샷 (rank, n1/n2/n3_score, n2_change, 리뷰수, 저장수)
- daily_inputs: 트래픽 투입 기록 (business_id, source_id, quantity, input_date)
- sources: 매체 (name, category, min_quantity, unit_price, grade)
- action_types: 행동유형
- alerts: 경고 (severity: red/yellow/green)
- crawl_logs: 크롤링 로그
- crawl_failed_items: 크롤링 실패 항목
- internal_data: 내부 데이터 (트래픽, 길찾기, 전화, 저장)
- market_prices: 시장가격
- advertiser_profitability: 수익성 분석
- memos: 로직메모
- audit_logs: 감사 로그
