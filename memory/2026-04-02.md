## 크몽 문의(Inbox) 크롤러 서비스명 추출 디버깅 (2026-04-02)

**현황:** `kmong-crawler/crawl-inbox.js` 스크립트에서 크몽 메시지 상세 페이지 내 "문의 서비스"에 연결된 서비스명을 정확하게 추출하지 못하고 있음. 현재 `(미확인)`으로 보고됨.

**시도된 방법:**
1.  초기에는 CSS selector (`li:has(strong:text("문의 서비스")) .service-item__title strong` 등)를 사용하여 서비스명 추출 시도. -> 실패.
2.  `page.evaluate`를 사용하여 DOM을 직접 탐색, "문의 서비스" `strong` 태그의 다음 형제 요소에서 텍스트 추출 시도. -> 실패. (이전 편집본과의 불일치 가능성 있음)

**현재 진행 중인 접근:**
-   `page.evaluate` 내에서 `document.querySelectorAll('strong')`를 사용하여 "문의 서비스" 라벨의 `strong` 태그를 찾고, 해당 요소의 다음 형제 요소를 순회하여 서비스명을 추출하는 로직을 재검토 및 구현 중.
-   또한, `[class*="message-card__content"] [class*="service-item__title"] strong` 셀렉터도 `page.evaluate` 내부에서 fallback으로 시도.

**다음 단계:** `crawl-inbox.js`를 다시 수정하여 `page.evaluate` 내부 로직을 더욱 견고하게 만들고, 재실행하여 서비스명 추출 성공 여부를 확인.
