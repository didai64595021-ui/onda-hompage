## 블로그 벤치마킹 시스템 (플레이스 블로그 추적기 그룹)

### 핵심 로직 확립
- **벤치마킹 방식**: 각 키워드 상위10위 중 N2 상승 경쟁사의 7일간 블로그 증가량 = 우리 권장 투입량
- AdLog `ajax_place_list.php` → N2 변동 + 블로그 변동 파싱 (data-id, data-name, data-rank 속성)
- 경쟁사 블로거 역추적: 네이버 블로그 검색 → 블로거ID → 총 글 수 확인
- iwinv 프록시(`http://49.247.137.28:3100`) 경유 네이버 크롤링

### AdLog API 정확한 경로
- 로그인: `https://adlog.kr/bbs/login_check.php` (mb_id/mb_password)
- 키워드 조회: `https://adlog.kr/adlog/ajax_place_list.php` (api_section=10, api_type=90, place_keyword, master_disp=1, place_dday=7)
- 응답: HTML 테이블 → regex 파싱 (data-id, data-name, data-rank, td[2]=블로그, td[4]=N2)

### 블로거 품질 판별 기준 (N2 상관관계 기반)
- ❌ 글 5개 미만 = 1회성 계정 → N2 역효과
- ❌ 업종 무관 잡다 리뷰 (아이폰수리+삼겹살 등) → N2 기여 0
- ❌ 리뷰한 업체의 N2가 하락 중인 블로거 → 효과 없음
- ✅ 글 10개+ & 업종 연관성 & 리뷰 업체 N2 상승 = 양질 블로거
- **핵심**: N2 올라가는 업체에 달린 블로거만 사용
