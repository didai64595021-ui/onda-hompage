# 네이버 크롤러 — 하이아이피 프록시 연동 + 2PC 운영 가이드

---

## 1. 하이아이피(HaiIP) 프록시 설정

### 1-1. 프록시 파일 만들기

하이아이피에서 발급받은 IP와 포트를 텍스트 파일로 저장합니다.

1. 하이아이피 클라이언트 실행 → 로그인
2. 프록시 상품에서 IP:PORT 목록 확인
3. 메모장으로 아래처럼 저장

**파일명**: `proxy_list.txt` (크롤러와 같은 폴더에 저장)

```
# 하이아이피 프록시 목록
# IP:PORT 형식 (한 줄에 하나씩)
123.456.789.10:8080
111.222.333.44:8080
55.66.77.88:3128
```

- `#`으로 시작하는 줄은 무시됩니다 (메모용)
- IP와 포트 사이는 `:` (콜론)으로 구분
- 프록시 개수 제한 없음 (많을수록 차단 위험 감소)

### 1-2. 프로그램에서 프록시 설정

프로그램 실행 후 설정 화면에서:

| 항목 | 설정값 |
|------|--------|
| 프록시 모드 | **프록시 사용** 선택 |
| 프록시 파일 | **proxy_list.txt** 경로 지정 |

또는 `crawler_config.json`을 직접 수정:

```json
{
  "proxy_mode": "프록시 사용",
  "proxy_file": "C:\\Users\\user\\Desktop\\비즈니스\\네이버 크롤러\\proxy_list.txt"
}
```

### 1-3. 프록시 동작 방식

- 프로그램이 자동으로 프록시 IP를 돌려가며 사용합니다
- IP 하나가 차단되면 자동으로 다음 IP로 전환됩니다 (5분 쿨다운)
- 차단된 IP는 5분 후 자동 복구됩니다
- Fingerprint(브라우저 정보)도 프록시마다 자동 변경됩니다

---

## 2. 프록시가 Selenium에도 적용됩니다 (업데이트)

기존에는 Selenium(순위 크롤링)에 프록시가 안 물렸는데,
이번 업데이트로 **Selenium도 프록시를 자동 사용**합니다.

- 프록시 파일이 설정되어 있으면 → Selenium 크롤링 시 프록시 IP 사용
- 프록시 없으면 → 기존대로 직접 연결

**이전 버전에서 "광명치과" 같은 키워드에서 Selenium 실패가 나던 이유:**
→ IP 차단 상태에서 Selenium이 프록시 없이 직접 연결하기 때문

**이번 업데이트 후:**
→ Selenium도 프록시 IP로 접속하므로 차단 확률 대폭 감소

## 3. 컴퓨터 2대로 돌리기 (2PC 운영)

### 3-1. 기본 원칙

- 2대가 **서로 다른 키워드**를 동시에 돌리면 됩니다
- 같은 키워드를 2대에서 동시에 돌리면 IP 차단이 빨라지므로 비추천
- **API 키는 공유 가능** (일일 25,000건 한도 공유)

### 3-2. 설정 방법

**PC 1 (기존 컴퓨터)**

`crawler_config.json`:
```json
{
  "proxy_mode": "프록시 사용",
  "proxy_file": "C:\\Users\\user\\Desktop\\비즈니스\\네이버 크롤러\\proxy_list.txt",
  "api_key1_id": "j2jMCfsKAStJ10rb0Q2A",
  "api_key1_secret": "nv10elHEYJ",
  "api_key2_id": "",
  "api_key2_secret": "",
  "keyword_queue": ["포항성형외과", "부산피부과"]
}
```

**PC 2 (새 컴퓨터)**

1. 크롤러 폴더 전체를 새 PC에 복사
2. `crawler_config.json`에서 **키워드만 다르게** 설정:

```json
{
  "proxy_mode": "프록시 사용",
  "proxy_file": "C:\\Users\\user\\Desktop\\비즈니스\\네이버 크롤러\\proxy_list.txt",
  "api_key1_id": "j2jMCfsKAStJ10rb0Q2A",
  "api_key1_secret": "nv10elHEYJ",
  "api_key2_id": "",
  "api_key2_secret": "",
  "keyword_queue": ["대구치과", "울산피부과"]
}
```

### 3-3. API 키 2개로 분리 (권장)

네이버 API 키를 2개 만들면 일일 한도가 2배로 늘어납니다.

1. https://developers.naver.com 접속 → 로그인
2. "Application" → "등록" → 검색 API (지역) 선택
3. 새 Client ID/Secret 발급

```json
{
  "api_key1_id": "기존키_ID",
  "api_key1_secret": "기존키_Secret",
  "api_key2_id": "새로운키_ID",
  "api_key2_secret": "새로운키_Secret"
}
```

- 키 2개 설정 시 프로그램이 자동으로 번갈아 사용합니다
- PC 1에 키1, PC 2에 키2로 분리해도 됩니다
- 같은 키를 2대에서 공유해도 됩니다 (한도만 공유)

### 3-4. 프록시도 분리 (권장)

하이아이피에서 IP를 여러 개 발급받으면:

**PC 1**: `proxy_list_pc1.txt`
```
100.100.100.1:8080
100.100.100.2:8080
100.100.100.3:8080
```

**PC 2**: `proxy_list_pc2.txt`
```
200.200.200.1:8080
200.200.200.2:8080
200.200.200.3:8080
```

→ 각 PC가 서로 다른 IP를 쓰므로 차단 확률 최소화

## 4. 자주 발생하는 문제 & 해결

### "Selenium 실패" 메시지가 뜹니다

| 원인 | 해결 |
|------|------|
| IP 일시 차단 | 프록시 설정 후 재시도. 하이아이피 IP 사용 시 해결됨 |
| Chrome 버전 불일치 | Chrome 최신 버전으로 업데이트 |
| VPS/서버에서 실행 | 반드시 실제 PC에서 실행 (서버는 봇으로 차단됨) |

→ Selenium 실패 시 자동으로 모바일 지도/API 방식으로 폴백하므로 데이터는 수집됩니다.
→ 단, 순위 정확도가 떨어질 수 있습니다.

### 특정 키워드만 안 긁어옵니다 (예: "광명치과")

- IP 차단은 키워드가 아니라 **시간 기반**입니다
- 동일 IP로 많이 크롤링하면 잠깐 차단됩니다
- 프록시 사용하면 IP가 계속 바뀌므로 해결됩니다
- 프록시 있는데도 안 되면: 딜레이를 1~3초로 늘려보세요

### "경기도 전체를 긁어옵니다"

- "광명치과"처럼 지역+업종 키워드 검색 시, 네이버가 해당 지역 결과가 부족하면 **자동으로 주변 지역까지 확장**합니다
- 이건 네이버 검색 자체의 동작입니다
- 프로그램은 결과를 **업종 우선 정렬**하므로 해당 업종이 상단에 옵니다
- 업종이 다른 업체는 하단에 별도 그룹으로 정리됩니다

## 5. 권장 설정값

| 항목 | 프록시 없을 때 | 프록시 있을 때 |
|------|---------------|---------------|
| 딜레이(최소) | 1.0초 | 0.3초 |
| 딜레이(최대) | 3.0초 | 2.0초 |
| 타임아웃 | 8초 | 8초 |
| 최대 페이지 | 3 | 5~10 |

프록시 사용 시 딜레이를 줄여도 차단 위험이 낮아 속도가 빨라집니다.

## 6. 파일 구성 (2PC 기준)

```
네이버 크롤러/
├── crawler_engine.py       ← 엔진 (수정된 버전으로 교체)
├── crawler_config.json     ← 설정 (PC마다 키워드 다르게)
├── proxy_list.txt          ← 하이아이피 프록시 목록
├── districts_data.json     ← 행정구역 데이터 (수정 불필요)
├── naver_cookies.json      ← 네이버 쿠키 (자동 생성)
└── 결과파일.xlsx           ← 크롤링 결과 (자동 생성)
```

---

## 7. 요약

1. 하이아이피에서 IP 발급 → `proxy_list.txt`에 저장
2. `crawler_config.json`에서 프록시 모드를 "프록시 사용"으로 변경
3. 2대 PC: 크롤러 폴더 복사 → 키워드만 다르게 설정
4. (권장) API 키 2개 발급 → PC별 분리
5. (권장) 프록시 IP도 PC별 분리

문의사항 있으시면 말씀해주세요!
